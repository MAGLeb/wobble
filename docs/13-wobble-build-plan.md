# WOBBLE — план сборки и открытые вопросы (08.07.2026)

Вторая игра под главный приз $15k. Лидер-кандидат, **гейтится спайком дня 1** (§3); при провале спайка — пивот на DIGSITE. Полная идеация и скоринг — [docs/12](12-second-game-ideation.md). Прошлое искусство (Tile Tower/Stonefall) и структурная оговорка «смотрю, а не играю» — там же.

## Замковый камень: модель провала (РЕШЕНО — гибрид со швом по слою)

Классический Stack и WOBBLE-обвал — **противоположные** модели провала (мягкий безвинный конец «нечем ставить» vs жёсткий именной «башня легла»). Склеены не механикой, а **слоями**:

- **Вход = Stack.** Скользящий блок над башней, ОДИН тап, знакомый кайф ловли качания. Это соло-ядро (зафиксировано пользователем как обязательное).
- **Блок НЕ режется** как двигатель провала (срез — только визуал/скоринг; рабочая ширина держится с полом). Наследовать Stack-сужение нельзя: оно эродирует общую ширину за ~20–40 неряшливых дропов и делает «последнего на тонком блоке» виновником незаслуженно.
- **Промах = блок ложится со СМЕЩЕНИЕМ** → копится офсет центра масс (крен).
- **Истина коллапса = скалярный «метр нестабильности»** (накопленный крен ≥ порога), НЕ честный Matter-топпл. Скаляр детерминирован → синхронен между устройствами, честен для блейма, тюнится под каденцию.
- **Matter/физика — ТОЛЬКО зрелищная анимация падения** поверх уже решённого скаляром исхода: один клиент считает падение → пишет кейфрейм-трек → все проигрывают запись, никто не ресимулирует.
- **Триггер «уронил» = центр масс всей башни вышел за footprint основания в момент ТВОЕГО дропа.** Единичный кривой дроп сам по себе не роняет; падает только уже накрененная башня. Виновник = чей дроп перевёл за порог.

Итог: тап знакомый, обвал и блейм настоящие, а техрисковый кросс-девайс детерминизм снят (истина — число + записанный реплей, не живая физика).

### Конкретная математика крена (РЕШЕНО 09.07)

Одно число **L** («крен»), старт 0, знак = сторона наклона.
- Каждый блок: считаем **промах `o`** (насколько мимо центра, signed — это и есть скилл; блок едет быстро, поймать центр трудно). **`L = L + o`.** Точный дроп крен не растит; дроп против наклона крен **уменьшает** (умелый игрок «спасает» башню — роль героя рядом с виновником).
- **Падение при `|L| ≥ Предел(высота)`**; Предел **сжимается с высотой** (выше = меньше терпит → сложность растёт сама). Виновник = чей блок толкнул за Предел.
- **АНТИ-ГРИФ (требование пользователя): потолок одного промаха `|o| ≤ o_max`, причём `o_max < Предел_min`.** Тогда один блок с ровной башни (L=0) в худшем случае даёт L=o_max < Предел → **уронить одним блоком физически нельзя.** Нужно ≥2 плохих дропа.
- **«Двое промазали → падает»**: два ужасных дропа подряд ≈ 2·o_max; на высокой башне (Предел мал) это > Предела → легла на втором; на низкой прощает. Малышку прощаем, взрослую башню два неряшливых дропа роняют.
- **Пример (для тюнинга):** Предел 10 на старте → сжимается к ~4 на высоте; o_max = 3. Ровная + 1 жуткий → L=3, стоит всегда. +ещё жуткий → L=6: низкая стоит, высокая падает.
- **Качание = анимация L** (чем больше |L|, тем сильнее шатается, видно ДО тапа — поэтому тап это навык). Каденция 60–120 блоков крутится распределением `o` (скорость/ширина свипа) + кривой Предела.
- Инструмент настройки: живая мини-песочница со ползунками (скорость блока / o_max / кривая Предела) — тюнить ощущение, не спорить на бумаге.

## §2. Решения пользователя (blocker) — дефолты, с которыми можно просто согласиться

1. **Модель провала** → гибрид выше.
2. **Что переживает обвал** → падает ВСЯ башня, архивируется «Башня #N» с рекордом высоты + мемориал, новая с нуля. «Пережил обвал» = имя было на башне в момент падения (леджер), не физическая персистентность блока. Частичный снос верха отвергнут (делает базу вечной → ломает каденцию).
3. **Тон вины** → инфамия, не позор. Мемориал называет виновника ИГРИВО: «Ты уронил Башню #14 на 87-м блоке» (событие-праздник). Флаир «Demolisher» — редкий желанный, без материального штрафа. Бюджет блоков одинаков за любой дроп (иначе все играют безопасно и драма умирает).
4. **Первый заход** → 2–3 коуч-марка поверх реального экрана + кнопка TRY (практика по желанию, не форсим — не жжём 3-мин бюджет судьи). В practice Предел ускорен → обвал гарантирован за ~30 с (судья видит драму, даже если общую башню уронить нельзя).
5. **Посев демо** → БЕЗ режиссуры «почти обвал» (решение пользователя 09.07: это попахивает подделкой). Вместо: башня ВСЕГДА видимо качается (опасность читается на любой высоте) + за пару дней демо реальная игра даёт высокую башню и пару настоящих обвалов → в ленте настоящие мемориалы с реальными никами (живой контент, не фейк-цифры). Правила — закреплённый коммент-объяснялка от бота в 2 строки (паттерн r/hotandcold). Судья ловит хук честно: качающаяся башня + мемориалы в ленте + пиннед-правила.
6. **Крючок возврата** → легаси + месть: «Башня #14, которую ты строил, пала — отстрой» / «твой блок теперь несущий, башня на 90 и качается».

## §3. Спайк дня 1 (09.07) — единственный настоящий убийца проекта

**СМЯГЧЁН (решение пользователя 09.07): пиксель-в-пиксель НЕ нужен.** Раз истина — скаляр L + позиции блоков (данные в Redis), согласованность события бесплатна; анимация падения у каждого может быть своя. Критерий — не визуальный детерминизм, а **согласованность СОСТОЯНИЯ и СОБЫТИЯ**.

**Критерий приёмки (ниже — пивот на DIGSITE, день заложен):**
- Ставлю блок → его позиция + новый L атомарно в Redis; ≥2 других клиента читают ТУ ЖЕ башню (совпадают данные, не пиксели анимации).
- Обвал: сервер/канон фиксирует событие (Башня #N легла на блоке K, виновник u/X) → realtime-сигнал «падаем» → у всех падение играет ~в один момент (анимация локальная, разная — ок).
- Turn-lock (watch/multi) держит 2 одновременных захвата (второй ждёт/отбивается).
- **Не зелено к вечеру 09.07 → пивот.** Это стандартный Redis+realtime, а не хардкорный физ-синхрон → техриск WOBBLE НИЖЕ, чем оценивалось; пивот куда менее вероятен.

Закрывается тем же спайком: формат канона (упорядоченный список `(x,y,rotation,ник)`, скорости не храним, дозапись в конец атомарно); формат обвала (кейфрейм-трек N×поза ~1.5 с рядом с мемориалом); wire-формат чужого хода (шлём ПАРАМЕТРЫ дропа: t0 спавна, скорость/направление свипа, таймстамп тапа → зритель анимирует локально → снап к rest; не пер-кадровый стрим); turn-lock lifecycle (SET NX с токеном, TTL 15 с + heartbeat — унифицировано с docs/14; краш/AFK → ход освобождается, незавершённый блок ОТБРАСЫВАЕТСЯ, частичное в канон не пишем); не-ронятельность прямой башни (масса/запас так, что прямую один плохой блок почти не роняет — замерить); зрительский слой MVP (лёгкая ставка «устоит?» на чужом ходу — валидировать вовлечение).

## §4. Дефолты (беру на себя)

- Видимый ридаут крена/опасности ДО тапа (индикатор + подрагивание) — превращает тап в навык, делает блейм честным.
- Идеальный дроп = рецентр (чуть снижает крен) + комбо-очки, без Stack-регроста ширины. Роль «спасителя» рядом с «виновником».
- Сложность растёт по высоте БАШНИ (уже/быстрее свип/сильнее качание), не по игроку.
- Каденция: большинство башен падают в окне **60–120 блоков** независимо от разброса скилла; финальный триггер — смещение блока игрока (вина адресна).
- Высота бесконечная, вехи-этажи каждые ~50 (именной чекпойнт = бенчмарк флаира), потолка нет.
- Логаут-игрок = соло-песочница (без ника, не трогает канон) + просмотр чужого хода + CTA «войди, чтобы поставить имя» в момент дропа.
- Кадрирование: камера крупно держит зону дропа + последние ~10–15 блоков; полный силуэт — мини-превью.
- **Экономика блоков (решение пользователя 09.07): регенерация 1 блок / ~3 часа** (энергия-модель), накопление с потолком ~3; новичок/судья стартует с 2–3 готовыми (иначе первая сессия без игры). Это сильнее «3/день»: возврат несколько раз в день + игрок вкладывается в ТУ ЖЕ башню и приходит проверить («мой блок ещё стоит? не уронили?»). Практика негейтед.
- **Математика крена — обкатана в песочнице-артефакте (v1):** L += промах, потолок промаха `o_max < Предел_min` = анти-гриф (один блок не валит), `2·o_max > Предел` = «двое неряшливых роняют». Дефолты из песочницы: o_max 30, Предел 96→42, полное сжатие к ~80 блокам. Тюнить финально там же.

## §4b. Визуальный язык (решение пользователя 09.07)

Дефолт-тема — **«Пиза/Рим»**: этажи = каменные аркадные ярусы (травертин, колонны/арки), башня растёт как классическая **Пизанская** — крен ⇒ буквально становится наклонной башней (идеальный тематический фит к механике). Тосканское небо, зелёная земля с фундаментом-плинтусом, кран опускает этаж (Tower Bloxx-референс). Камера **по режимам** (уточнено красной командой 09.07): idle/cover и collapse — **низ на земле, вся башня в кадре** (решение пользователя); BUILD — **крупный план зоны дропа** + мини-силуэт (на телефоне в 80-этажном полном виде скользящий этаж = пиксельная полоска, тап-скилл нечитаем). Качание — **только высоко И при крене** (короткая башня стоит смирно); живость cover без фейк-качания — облака/флажок/ghost-реплей. **Тема-переключатель — будущий апдейт.** Обкатано в песочнице-артефакте.

## §5. Можно отложить (не блокирует MVP)

Полный сервер-сайд античит (клиенту доверяем + дешёвый санити-чек); очередь ходов в пик (при 5–50 доска ≈свободна); тонкая настройка каденции под саб (офлайн день 2); Devvit-пуш (подключить к дню 11); расширенный «Demolisher»-леджер.

## Где реально живёт риск

Убить проект целиком может только спайк §3. §2 — соглашательские решения на 10 минут. Второй риск — судья за 3 мин не увидит обвал — закрывается посевом (§2.5), не механикой, поэтому не откладывается.
