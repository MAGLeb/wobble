## 1. Язык, тема, камера

- **Весь продукт на английском.** «Tower #14 fell at storey 87 — toppled by u/X».
- Тема «Пиза» (docs/13 §4b): каменные аркадные этажи, качание только высоко+крен (короткая башня на земле стоит смирно — физическая правда).
- **Камера по режимам (устранён конфликт docs/13↔14):** BUILD — **крупный план зоны дропа** (последние ~10–15 этажей; иначе на телефоне в 80-этажной башне скользящий этаж = полоска пикселей, тап-скилл нечитаем) + мини-силуэт всей башни сбоку; **idle/cover и collapse-реплей — полный вид** «низ на земле, вся башня в кадре» (как просил пользователь).
- **Живой cover без фейк-качания:** короткая башня не качается, но экран не статичен — дрейф облаков, флажок на кране, ghost-реплей последних 2–3 дропов при загрузке, строка «storey 43 by u/X» из канона (ts есть; свежесть скрываем при stale), чип-кладбище «Tower #15 · 14 have fallen» (тап → мемориал-карточки из архива).
- **Phaser не используем** (канвас-порт песочницы; двойной билет $5k осознанно отменён). Падение — покадровая канвас-анимация как база (не «fallback»).

## 2. Архитектура (Devvit Web)

**devvit.json (поправлено по докам):** `post.entrypoints` (`default`, height `tall`); `server.entry`; `permissions: { redis: true, realtime: true }` (дефолт false!); `permissions.reddit.scope: moderator` — вероятно нужен для setUserFlair (проверить в спайке; это НЕ asUser, но строка на ревью); `/api/*` — обычные роуты сервера (в devvit.json не декларируются), `/internal/*` — только для cron/triggers (ядру cron не нужен).

**Turn-lock (переведён с watch/multi на SET NX — атомарно и проще):**
- claim: `set('turn_lock', {user, token: random, expiresAt}, {nx: true, expiration: TTL 15 с})` → успех = ход мой; отказ = «u/X строит» + **FIFO-слот «you're next»** (один, с автоклеймом после освобождения — иначе судья при наплыве не построит за 3 мин).
- drop: `watch('turn_lock','tower:current')` → прочитать, **сверить token** (не верить TTL) → `multi`: канон, del лока, энергия −1, статы, при обвале инкремент `tower:counter` → `exec`; null → retry.
- **Side-эффекты (мемориал-пост, флаиры, realtime) — строго ПОСЛЕ успешного exec** (иначе при retry два мемориала).
- Heartbeat продлевает лок, пока игрок целится; TTL истёк → ход свободен, энергия не сгорела, недо-ход отброшен.

**Redis:** `tower:current` — JSON `{id, blocks:[{dx,u,ts}], L, createdAt}` (~5–10 KB на сотни блоков, лимит 5 MB — ок; сериализация через лок = гонок нет) · `tower:counter` · `tower:archive:{id}` · `energy:{uid}` `{n,ts}` лениво, потолок 3, новичок 3 · `stats:{uid}` `{placed,survived,caused}`. Cleanup: PostDelete-триггер чистит ключи, expire на user-данные (перед publish).

**Realtime (канал `tower_{postId}` — двоеточие в имени запрещено):** `turn_claimed` · `block_placed {dx,u,L,h}` · `collapse {id,h,culprit}` · `tower_new`. Доставка не гарантирована → в каждом сообщении монотонный `h`; клиент при гэпе и onConnect рефетчит `/api/state`. Зрители анимируют чужой дроп по параметрам (слайд→посадка), ожидающий в слоте видит чужой слайд вживую.

**postData (2 KB, первый кадр <1 c):** `{towerId, h, L, lastUser, fallenCount}`; **обновлять при каждом дропе** через `post.setPostData` (затирает целиком → мерджить!), иначе обложка замрёт на нуле.

**Мемориал:** submitCustomPost от апп-аккаунта из дроп-хендлера (подтверждено докой, runAs не нужен) открывается ТЕМ ЖЕ webview → **memorial-режим клиента** по `postData {type:'memorial', towerId}` (карточка + архивная башня). Аварийный вариант — обычный текстовый пост. Решить формат к 10.07 → берём memorial-режим.

**Логаут:** practice + watch, CTA в момент тапа.

## 3. Экраны/состояния

1. **Cover/watch:** полная башня (живая — §1), высота, кладбище-чип, BUILD / TRY.
2. **Build:** крупный план, скользящий этаж, тап; фидбек «Storey 44 · yours forever».
3. **Queue-слот:** «you're next» + чужой слайд вживую.
4. **Practice:** соло, водяной знак, **ускоренный Предел — обвал ГАРАНТИРОВАН за ~30 с** («построил → закачалась → легла» — судья, не сумевший уронить общую башню (свежая: 3 ужасных дропа = L 90 < Предела 96!), увидит драму здесь).
5. **Out of blocks:** «next block in 2:47 · keep practicing» → авто-маршрут в practice (иначе сессия судьи мертва на 90-й секунде).
6. **Collapse:** полный вид, покадровое падение → карточка «Tower #14 · 87 storeys · toppled by u/X · top builders» → «Tower #15 begins».
7. **Memorial-режим** (отдельный пост): та же карточка из архива.
8. **Onboarding:** 2–3 коуч-марка поверх реального экрана (тап по центру / крен-бар / имя на этаже). Практику НЕ форсим — кнопка TRY всегда есть.
